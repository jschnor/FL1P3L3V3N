Standard Installation

This section is for standard HTML websites. For WordPress/CMS installations, see WordPress Installation. Hyena has a few required files. Installation is only a matter of uploading the following files to your server and linking them on your page.

  • jQuery
  • Hyena Plugin File (JavaScript)
  • Hyena CSS File

Step 1: Upload the Hyena Folder to your site's root folder. This is typically where your index.html file resides.

Step 2: Link the Hyena CSS file in the <HEAD> section after your stylesheet. You can copy the following: <link rel="stylesheet" href="Hyena/jquery-Hyena-1.2.min.css" type="text/css" media="all" /> It's recommended to place this after your other CSS Links, to avoid overwriting Hyena styles.

Step 3: If your website does not have jQuery installed, link the jQuery file in the <HEAD> section of your page.
<script type="text/javascript" src="Hyena/js/jquery-1.9.1.min.js"></script> To check if jQuery is already intalled on your site, look in the <HEAD> section of your page for a <SCRIPT> tag with 'jquery-x.x.x.xxx.js' in the src attribute (Where x would indicate a version number). jQuery is a JavaScript library that provides a set of tools which Hyena uses. Since parts of Hyena rely on these components, it's important that jQuery is loaded before the Hyena Javascript file. It's optimal to place jQuery before all other <SCRIPT> links.

Step 4: Link the Hyena JavaScript Plugin File. <script type="text/javascript" src="Hyena/jquery-Hyena-1.2.min.js"></script> Hyena should be loaded after jQuery.

Step 5: Last, confirm that all files have been correctly linked. Open Firefox (Firefox is recommended for this step). Navigate to your website. View the source code by pressing CTRL-U (CMD-U on Mac). A window will open revealing the source code. At the top - inside the <HEAD> section of the page, you'll see all of the links you've added. When linked correctly, clicking the link should navigate to the file contents. If the file contents don't appear, the browser can't find the file at the path you specified.

Complete Code: Below is the correct loading order. You can also refer to the included demos for more examples.

<HEAD>
...
<link rel="stylesheet" href="Hyena/Hyena-1.2.min.css" type="text/css" media="all" />
<script type="text/javascript" src="Hyena/jquery-1.9.1.min.js"></script>
<script type="text/javascript" src="Hyena/jquery-Hyena-1.2.min.js"></script>
...
</HEAD>

Containers

Before we dive into making different effects, it helps to understand what Hyena is doing. Hyena is based around the simple idea that nearly every website animation effect can be achieved with 2-3 simple HTML containers. This approach can actually be more efficient and lightweight than writing custom code specific to each effect. When you call Hyena on an element, it builds those animation containers around that element. To the right is a diagram of what elements are made and how they interact with each other. Understanding this diagram will help when setting up your effects.

Container Your element is wrapped inside of the Hyena Container (outlined in black). Background The Background element (shown in gray) is placed inside of the Hyena Container. This example shows the Background positioned like a Caption. The position of the Background Container is relative to the Container and a Coordinate System is used to position the Background. Basically, 0,0 will position the Background at the top-left corner of the Container.
Later you will learn that the "coords" setting controls the movement of this container. Text Container The Text Container (shown in Blue) is positioned inside of the Background Container and can also be animated independently.
Later you will learn that the "text_coords" setting controls the movement of this container. Effects use a different combination of the above containers

HTML Setup

The power of Hyena revolves around the fact that you build your HTML and CSS the way you are already used to. The majority of examples in the documentation and the tutorials are used on images, but you can use any HTML Element you want. <img src="path/to/image.png" /> There isn't anything special about the above code - just a simple image tag. In order for us to add a Hyena Caption to this image, we have to be able to target it, so we give it a class: <img src="path/to/image.png" class="Hyena_Caption" /> Now that this image has a class of "Hyena_Caption", we have a way of finding it on the page. This is important, and will be used in the next section.
You can apply the same class to multiple elements. Text Text is added by using the ALT attribute: <img src="path/to/image.jpg" class="Hyena_Caption" alt="My Caption Text Here"/> Links Links are added using the REL attribute: <img src="path/to/image.jpg" class="Hyena_Caption" rel="http://www.codecanyon.net"/> You can further control linking using the link_control setting. This setting has 2 values. "link_control" : "true,_blank" The first value allows for BIG linking. BIG linking expands the coverage of your link from the Hyena Background Container to the main Container. For example, with Big Linking disabled, the link on a Caption would be around the Caption itself. With Big Link enabled, the link would wrap the entire image (or element). The second option allows you to control how the link is opened. To open the link in a new tab use "_blank", otherwise use "_self". Positioning the Element Hyena will automatically build all of the Containers covered in the previous section - around your element(s). However, if you want to position your element, use a wrapper and apply all of the positioning CSS to the wrapper instead of the element itself. For example, if you want to add a 20px margin to the top of your thumbnail image: <div style="margin-top:20px;">
<img src="path/to/image.png" />
</div>

Calling Hyena

Plugin Call Once the HTML is setup, a simple bit of JavaScript calls Hyena into action. This can be placed anywhere on your page, however it is recommended to place all of your Hyena Plugin Calls inside a single <script> tag at the end of your <HEAD>: <script>
$(window).load(function(){
$('.Hyena_Caption').Hyena();
});
</script> The above code reads easily. First, we setup an event: $('window').load(function(){ ... }); This events runs some code when the page has finished loading. The code we are running is: $('.Hyena_Caption').Hyena(); When the window loads, the browser searches the page for any HTML elements that have the class of "Hyena_Caption". Hyena is called on all elements found with that class. Lastly, we need to add some settings to tell Hyena what we want it to do with the found element(s). Passing in settings is quite easy - and we will cover all settings in depth shortly. For now we just want the basic settings to turn this into a Caption: <script>
$(window).load(function(){
$('.Hyena_Caption').Hyena({
"mode" : "caption",
"coords" : "0,0||0,0||0,0",
"text_coords" : "0,0||0,0||0,0"
});
});
</script> Do not place a comma after the last setting in the Hyena Call At this point we are only focused on calling Hyena - not setting up a working Caption. You should only be concerned with:

  • Understanding how to target HTML Elements
  • Calling Hyena on those Elements
  • Passing in the customization Settings
  • Do not place a comma after the last setting

WordPress Considerations If you're using WordPress, everything is done exactly the same... except for one minor thing:
the $ becomes jQuery: <script>
jQuery(window).load(function(){
jQuery('.Hyena_Caption').Hyena({
"mode" : "caption",
"coords" : "0,0||0,0||0,0",
"text_coords" : "0,0||0,0||0,0"
});
});
</script> Common Mistake #1: Class vs ID There a many ways to call Hyena on a single element or multiple elements. Version 1.2 has seen some considerable updates - including a new tiered customization level system - which is covered in the next section. Basically, you can customize a group of elements in the plugin call, then drill down and customize the individual elements even more. To take full advantage of this system it is important to understand the differences between IDs and Classes.

IDs are used for only 1 element on a page. They are unique. If you are setting up a Hyena effect on 1 element only, use an ID. IDs provide the best performance.
Do not use IDs more than once per page.
Classes are used for targeting many elements at once.
Common Mistake #2: Multiple Hyena Calls You can load as many Hyena calls as you need, however it is best to use a bit of strategy and break them up based on effect type: <script>
$(window).load(function(){
$('.Hyena_Caption').Hyena({ ... });
$('.Hyena_Tooltip').Hyena({ ... });
$('.Hyena_Panel').Hyena({ ... });
$('.Hyena_Lightbox').Hyena({ ... });
});
</script> Note that all calls are inside a single <SCRIPT> tag and within the same $(window).load() event.

Configuring Movement

Background Container Movement Configuring Background Container movement is done with one setting: "coords" : "StartX,StartY || EndX,EndY || ReturnX,ReturnY" The values you enter in this setting are percentages using a 0,0 Coordinate System. 0,0 represents the top-left corner of the element, while 100,100 represents the bottom-right corner. The "coords" setting controls the animation path for the Hyena Background Container. For Captions, these are the blue boxes that pop up.

To have the blue Caption box come in from the bottom and return to the bottom: "coords" : "0,100 || 0,75 || 0,100" To make the Caption box come in the right and return to the right: "coords" : "100,0 || 0,0 || 100,0" Text Container Movement Controlling the animation for the Text is done exactly the same way, using the "text_coords" setting: "text_coords" : "StartX,StartY || EndX,EndY || ReturnX,ReturnY" To have the Caption Text come in from the top and return to the top: "coords" : "0,-50 || 0,10 || 0,-50" To make the Caption Text come in the left and return to the right: "coords" : "-50,0 || 10,0 || 100,0"

Styling Hyena Effects

Styling Hyena animations is done by dynamically assigning classes to the Hyena Background Container and the Hyena Text Container.

To add a class to the main Container, use: "container_class" : "className" To add a class to the Background, use: "bg_class" : "className" To add a class to the Text, use: "text_class" : "className" To make a Caption Background black: "bg_class" : "captionBackground"

.captionBackground { background-color:#000000; } Making it Responsive The Background Container class is where you define the dimensions for most of your effects. The examples that came in the download assign classes to the Hyena effects. Making Hyena effects responsive is done using your existing media queries in your theme / template. For example, if you create a Caption effect, and apply the dimensions using: "bg_class" : "captionBackground"

.captionBackground { width:200px; height:50px; background-color:#000000; } The above code applies the .captionBackground class to the Caption Effect's Background Container. In that class you have defined the dimensions for the Caption Background Container. This is the blue component in the demo Caption effects. If the image dimensions change for a tablet size, you may also need to update the Caption Background container dimensions to match. Locate the appropriate media query and use the same class - with any required modifications: @media only screen and (min-device-width : 768px) and (max-device-width : 1024px){
.captionBackground { width:Xpx; height:Ypx; background-color:#000000; }
}

Customization Levels

At this point you have seen how to initialize Hyena on multiple elements using classes.

The following example creates 4 thumbnails: <img src="path/to/image1.png" class="Hyena_Caption" alt="Caption 1"/>
<img src="path/to/image2.png" class="Hyena_Caption" alt="Caption 2"/>
<img src="path/to/image3.png" class="Hyena_Caption" alt="Caption 3"/>
<img src="path/to/image4.png" class="Hyena_Caption" alt="Caption 4"/> Next, we convert all of these into Captions using a single Hyena Call: <script type="text/javascript">
$(window).load(function(){
$('.Hyena_Caption').Hyena({
"coords" : "0,100||0,75||0,100",
"text_coords" : "0,50||0,5||0,50",
"bg_class" : "captionBackground",
"text_class" : "captionText"
});
});
</script> You should be familiar with what is going on above. The Call is running Hyena on all elements with class="Hyena_Caption". The "coords" setting tells Hyena that the Background Container will be moving in from the bottom of the element. The "text_coords" setting tells Hyena how to animate the text. Lastly, the "bg_class" and "text_class" settings apply classes to the Background and Text Containers, respectively.

All of the images receive the captionBackground class on the Hyena Background Container and the captionText class on the Hyena Text Container. However, we can customize this further - at the individual element level - using the data-Hyena attribute. data-Hyena We can isolate a single thumbnail image, and customize the animation effect a bit more using the data-Hyena attribute: <img src="path/to/image1.png" class="Hyena_Caption" alt="Caption 1"
data-Hyena='{ "bg_class" : "specialCaption" }'
/>
 Now the Background Container for the first Caption will receive the specialCaption class, rather than the captionBackground class. All settings placed in the data-Hyena attribute will override settings in the main plugin call. This pattern effectively allows you to configure a large group of items quickly (in the main Plugin Call), then "clean up" customization with a couple settings here and there - cutting down on the number of settings required.

The Examples/caption_multiple.html uses the data-Hyena attribute to change the color of the Caption Backgrounds. All Hyena settings can be used in the main Plugin Call and in the data-Hyena attribute.

Target Containers

Target Containers are only used for certain effects such as Sliding Panels and Lightboxes. Basically, when an animation occurs somewhere other than where the user interacts - a Target Container is used. For Sliding Panels, this is the main Container for the Sliding Panel. For Lightboxes - the Target Container is the Lightbox itself. "target_container" : "className" The Target Container setting is used for one purpose. It provides a way for you to assign a class to the Target Container. Say you wanted to add a Border to the bottom of your Sliding Panel. Using the target_container setting, allows you to assign a class to the Sliding Panel container, then quickly add a border to it. "target_container" : "slidingPanel1"

.slidingPanel1 { border-bottom:5px solid #000000; } Refer to the Sliding Panel examples for uses of Target Containers.

Injecting HTML

For more advanced cutomization scenarios, you can write your own HTML and inject it into any Hyena animation you like. Here's a bit of code you have just written on your page: <div id="myDIV">
<h1>My Awesome HTML</h1>
<p>This is some text.</p>
</div> Now, we can setup a Sliding Panel that opens when we click an image. <img src="path/to/image.png" class="Hyena_Panel"/>

...
<script type="text/javascript">
$(window).load(function(){
$('.Hyena_Panel').Hyena({
"coords" : "0,-50||0,0||0,-50",
"text_coords" : "0,0||0,0||0,0",
"bg_class" : "slidingPanelBackground",
"target_container" : "slidingPanel",
"html_mode" : "#myDIV",
"on_click" : true,
"mode" : "panel"
});
});
</script> In the above code, we have setup a image with class="Hyena_Panel". Then inside the Hyena plugin call we setup the animation coordinates to slide the panel down from the top of the screen. We assign the "slidingPanelBackground" class to the Background Container. The main Sliding Panel Container (Target Container) receives the class "slidingPanel". We set "on_click" to true to fire the panel when clicking on the thumbnail, and we set the animation mode to "panel".

Finally, you'll notice that we have taken the #myDIV block of code that we wrote earlier and injected it into the animation effect using the "html_mode" setting. Animating Inside Your HTML After you have injected your block of HTML, you also configure it to allow Hyena to animate your block of HTML. Basically, what we do is trick Hyena into thinking that the block of HTML you just injected into the animation is actually the Hyena Text. This is being done in the last Sliding Panel example on the demo page. <div id="myDIV">
<p class="hv_text">
<h1>My Awesome HTML</h1>
<p>This is some text.</p>
</p>
</div> By adding the class="hv_text", Hyena will treat your animation container like Hyena text - allowing you to configure the animation path for your block of HTML just like you would with Text - using the "text_coords" setting. The hv_text must be used on a <p> tag. Special Links When injecting HTML, you can add a link using the hv_link class: <div id="myDIV">
<p class="hv_text">
<h1>My Awesome HTML</h1>
<p>This is some text.</p>
</p>
<a class="hv_link"> href="http://www.codecanyon.net"></a>
</div> Closing Target Containers There are a couple quick classes you can apply to elements inside your block of injected HTML that Hyena will recognize.

Add the following class to any element to convert it into a button to close a Lightbox: class="close_lightbox" Add the following class to any element to convert it into a button to close an Alert: class="close_alert" Add the following class to any element to convert it into a button to close a Sliding Panel: class="close_panel"

<script type="text/javascript">
$(window).ready(function(){
$('img.hyena').Hyena({ ... });
});
</script>

WordPress Users: Replace every instance of $ with jQuery

Let's break this down a bit. First, we're setting up an event:
$(window).ready(function(){ ... });
Next, when the browser is ready, we're calling Hyena:
$('img.hyena').Hyena({ ... }); In the HTML section, you added a class to your images. Inside $('.hyena').Hyena({ ... }); the . . . is where optional Hyena settings go. The $('img.hyena') is called the selector. It is exactly the same as the class name of your image(s).

Passing in Settings

So far, the plugin call is setup and you have assigned the "hyena" class to all GIFs that you want to have controls. Hyena has now selected all of your GIFs with class="hyena" and loaded the default Hyena settings. You can now customize all of these images inside the plugin call (discussed on the previous page) using settings. Loading a Skin You can load Skins for all of the GIFs using the player_style setting. Hyena comes with 6 Skins.

<script type="text/javascript">
$(window).ready(function(){
$('img.hyena').Hyena({
"player_style" : 2
});
});
</script>

Individual Customization Hyena allows you to pass in settings globally - to affect all Hyena Animated GIFs (as shown above) as well as individually. For example, you can use the player_style setting in the plugin call to load Skin 2 (as shown above). This loads Skin 2 for all GIFs. You can then find a single GIF and load Skin 3 using the player_style setting directly on it.
<img src="my/funny/GIF.gif" class="hyena" data-hyena='{ "player_style" : 3 }' /> Adding settings in the data-hyena attribute is exactly the same as adding settings in the plugin call. Never place a comma after the last setting

Hyena Options

Controls / Buttons By default, player controls are enabled. You can disable player controls using: "controls" : false When the controls are disabled, Animated GIFs become playable when clicked. Button graphics are typically loaded based on the Skin (player_style) that is loaded. However, this can be overriden using: "control_style" : 3 To hide the Player Control when the page loads, use: "show_button" : false You can also control the size of the player controls as a size percentage of the animated GIF, using: "button_size" : "20%" To set the Fade In/Out opacities of the player controls, use: "control_opacity" : "0,0.5" The speed of the player control fade animation can be set using (in milliseconds): "fade_speed" : "250,250" When the Animated GIF initially loads, they will fade in. You can control the speed using: "player_fade_speed" : 350 Events When controls are disabled, the Animated GIF interaction defaults to click. Hyena GIFs can also be activated on hover using: "on_hover" : true You can set Hyena Animated GIFs to Play and Stop with Timers (in milliseconds), using: "on_timer" : "stopTime,playTime" For example, if you wanted the GIF to Play for 5 seconds then Stop for 3 seconds: "on_timer" : "3000,5000"
